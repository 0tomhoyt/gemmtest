# UniGEMM Fuzz Test — AI辅助驱动的工程质量实践

## 项目简介

UniGEMM Fuzz Test 是一个面向高性能计算领域的矩阵乘法（GEMM）数值正确性验证框架。项目实现了18阶段自动测试编排，覆盖3种浮点精度（FP32/FP16/BF16）、3种矩阵维度范围、单线程与多线程两种BLAS模式，并支持macOS与ARM64服务器的跨平台部署。整个项目在Claude Code辅助下完成架构设计、编码实现和持续迭代。

## AI辅助开发中的工程质量实践

本项目的核心价值在于：**AI不仅加速了开发，更在工程质量维度带来了传统开发模式难以实现的提升。**

**测试即设计约束。** 项目内建了一套完整的"测试自验证"机制——包含故意注入缺陷的buggy实现，用于反向验证测试框架本身的有效性。这种"用错误代码验证正确代码"的工程思路，是在与Claude Code的交互过程中逐步形成的，AI协助快速实现了buggy版本的精确缺陷注入与独立参考实现的双路对比架构。

**渐进式质量收敛。** 开发过程中，另一个AI代码审查工具对项目进行了独立审查，发现了leading dimension未做随机padding、macOS平台头文件兼容性、FP16次正规数处理等实质性工程问题。这些问题随后由Claude Code在分钟级别完成定位与修复，形成了"AI开发 → AI审查 → AI修复"的高效质量闭环。这种多AI工具交叉验证的模式，确保了单一AI工具的盲区能被另一个工具捕获。

## 成果与意义

项目最终实现300次模糊测试全部通过，buggy实现被准确检出。18阶段的失败报告支持精确到stage级别的错误归因，体现了AI辅助下对可观测性工程的重视。这一实践表明：**AI辅助开发的价值不仅在于速度，更在于通过多工具协作建立可持续的质量保障机制**——当AI同时参与代码生产和质量验证时，工程标准的执行门槛被显著降低，而质量水位反而得到提升。
